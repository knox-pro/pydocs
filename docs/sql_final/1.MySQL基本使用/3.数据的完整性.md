在表中为了更加准确的存储数据，保证数据的正确有效，可以在创建表的时候，为表添加一些强制性的验证，包括数据字段的类型、约束

数据类型： 整数：int

![](http://cyc.chenyangstudy.cn/book-sql/chapter01/img/10.png)

小数：decimal

decimal表示浮点数，如decimal(5,2)表示共存5位数，小数占2位

字符串：varchar,char

![](http://cyc.chenyangstudy.cn/book-sql/chapter01/img/11.png)

日期时间: date, time, datetime

![](http://cyc.chenyangstudy.cn/book-sql/chapter01/img/12.jpg)

约束：

主键primary key：物理上存储的顺序

非空not null：此字段不允许填写空值

惟一unique：此字段的值不允许重复

默认default：当不填写此值时会使用默认值，如果填写时以填写为准

外键foreign key：对关系字段进行约束，当为关系字段填写值时，会到关联的表中查询此值是否存在，如果存在则填写成功，如果不存在则填写失败并抛出异常，使用外键在数据量很大的时候会影响数据库的性能。
