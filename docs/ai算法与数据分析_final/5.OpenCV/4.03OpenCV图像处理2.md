## 边缘检测

### 1 原理

边缘检测是图像处理和计算机视觉中的基本问题，边缘检测的⽬的是标识数字图像

中亮度变化明显的点。图像属性中的显著变化通常反映了属性的重要事件和变化。

边缘的表现形式如下图所示：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/61_1.jpg)

图像边缘检测⼤幅度地减少了数据量，并且剔除了可以认为不相关的信息，保留了

图像重要的结构属性。有许多⽅法⽤于边缘检测，它们的绝⼤部分可以划分为两

类：基于搜索和基于零穿越。

基于搜索：通过寻找图像⼀阶导数中的最⼤值来检测边界，然后利⽤计算结果

估计边缘的局部⽅向，通常采⽤梯度的⽅向，并利⽤此⽅向找到局部梯度模的

最⼤值，代表算法是Sobel算⼦和Scharr算⼦。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/61_2.jpg)

基于零穿越：通过寻找图像⼆阶导数零穿越来寻找边界，代表算法是Laplacian

算⼦。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/62_1.jpg)

### 2 Sobel检测算⼦

Sobel边缘检测算法⽐较简单，实际应⽤中效率⽐canny边缘检测效率要⾼，但是边

缘不如Canny检测的准确，但是很多实际应⽤的场合，sobel边缘却是⾸选，Sobel

算⼦是⾼斯平滑与微分操作的结合体，所以其抗噪声能⼒很强，⽤途较多。尤其是

效率要求较⾼，⽽对细纹理不太关⼼的时候。

#### 2.1 ⽅法

#### 2.2 应⽤

利⽤OpenCV进⾏sobel边缘检测的API是：

Sobel_x_or_y = cv2.Sobel(src, ddepth, dx, dy, dst, ksize, scale, delta, borderType)

参数：

src：传⼊的图像

ddepth: 图像的深度

dx和dy: 指求导的阶数，0表示这个⽅向上没有求导，取值为0、1。

ksize: 是Sobel算⼦的⼤⼩，即卷积核的⼤⼩，必须为奇数1、3、5、7，默认

为3。

注意：如果ksize=-1，就演变成为3x3的Scharr算⼦。

scale：缩放导数的⽐例常数，默认情况为没有伸缩系数。

borderType：图像边界的模式，默认值为cv2.BORDER_DEFAULT。

Sobel函数求完导数后会有负值，还有会⼤于255的值。⽽原图像是uint8，即8位⽆

符号数，所以Sobel建⽴的图像位数不够，会有截断。因此要使⽤16位有符号的数

据类型，即cv2.CV_16S。处理完图像后，再使⽤cv2.convertScaleAbs()函数将其

转回原来的uint8格式，否则图像⽆法显示

Sobel算⼦是在两个⽅向计算的，最后还需要⽤cv2.addWeighted( )函数将其组合起

来

Scale_abs = cv2.convertScaleAbs(x) # 格式转换函数

result = cv2.addWeighted(src1, alpha, src2, beta) # 图像混合

示例：

    # 5 图像显示

### 4 Canny边缘检测

Canny 边缘检测算法是⼀种⾮常流⾏的边缘检测算法，是 John F. Canny 于 1986

年提出的，被认为是最优的边缘检测算法。

4.1 原理

Canny边缘检测算法是由4步构成，分别介绍如下：

第⼀步：噪声去除

由于边缘检测很容易受到噪声的影响，所以⾸先使⽤\\(5*5\\)⾼斯滤波器去除噪

声，在图像平滑那⼀章节中已经介绍过。

第⼆步：计算图像梯度

对平滑后的图像使⽤ Sobel 算⼦计算⽔平⽅向和竖直⽅向的⼀阶导数（Gx 和

Gy）。根据得到的这两幅梯度图（Gx 和 Gy）找到边界的梯度和⽅向，公式如下:

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/65_2.jpg)

如果某个像素点是边缘，则其梯度⽅向总是垂直与边缘垂直。梯度⽅向被归为四

类：垂直，⽔平，和两个对⻆线⽅向。

第三步：⾮极⼤值抑制

在获得梯度的⽅向和⼤⼩之后，对整幅图像进⾏扫描，去除那些⾮边界上的点。对

每⼀个像素进⾏检查，看这个点的梯度是不是周围具有相同梯度⽅向的点中最⼤

的。如下图所示：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/66_1.jpg)

A点位于图像的边缘，在其梯度变化⽅向，选择像素点B和C，⽤来检验A点的梯度

是否为极⼤值，若为极⼤值，则进⾏保留，否则A点被抑制，最终的结果是具有“细

边”的⼆进制图像。

第四步：滞后阈值

现在要确定真正的边界。 我们设置两个阈值： minVal 和 maxVal。 当图像的灰度

梯度⾼于 maxVal 时被认为是真的边界， 低于 minVal 的边界会被抛弃。如果介于

两者之间的话，就要看这个点是否与某个被确定为真正的边界点相连，如果是就认

为它也是边界点，如果不是就抛弃。如下图：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/67_1.jpg)

如上图所示，A ⾼于阈值 maxVal 所以是真正的边界点，C 虽然低于 maxVal 但⾼

于 minVal 并且与 A 相连，所以也被认为是真正的边界点。⽽ B 就会被抛弃，因为

低于 maxVal ⽽且不与真正的边界点相连。所以选择合适的 maxVal 和 minVal 对于

能否得到好的结果⾮常重要。

4.2 应⽤

在OpenCV中要实现Canny检测使⽤的API:

canny = cv2.Canny(image, threshold1, threshold2)

参数：

image:灰度图，

threshold1: minval，较⼩的阈值将间断的边缘连接起来

threshold2: maxval，较⼤的阈值检测图像中明显的边缘

示例：

## 模版匹配和霍夫变换

### 1 模板匹配

#### 1.1 原理

所谓的模板匹配，就是在给定的图⽚中查找和模板最相似的区域，该算法的输⼊包

括模板和图⽚，整个任务的思路就是按照滑窗的思路不断的移动模板图⽚，计算其

与图像中对应区域的匹配度，最终将匹配度最⾼的区域选择为最终的结果。

实现流程：

准备两幅图像：

1.原图像(I)：在这幅图中，找到与模板相匹配的区域

2.模板(T)：与原图像进⾏⽐对的图像块

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/70_1.jpg)

滑动模板图像和原图像进⾏⽐对：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/70_2.jpg)

将模板块每次移动⼀个像素 (从左往右，从上往下)，在每⼀个位置，都计算与模板

图像的相似程度。

对于每⼀个位置将计算的相似结果保存在结果矩阵（R）中。如果输⼊图像的

⼤⼩（WxH）且模板图像的⼤⼩(wxh)，则输出矩阵R的⼤⼩为（W-w + 1,H-h

  * 1）将R显示为图像，如下图所示：


![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/71_1.jpg)

获得上述图像后，查找最⼤值所在的位置，那么该位置对应的区域就被认为是

最匹配的。对应的区域就是以该点为顶点，⻓宽和模板图像⼀样⼤⼩的矩阵。

#### 1.2 实现

我们使⽤OpenCV中的⽅法实现模板匹配。

API：

res = cv.matchTemplate(img,template,method)

参数：

img: 要进⾏模板匹配的图像

Template ：模板

method：实现模板匹配的算法，主要有：

  1. 平⽅差匹配(CV_TM_SQDIFF)：利⽤模板与图像之间的平⽅差进⾏匹配，


最好的匹配是0，匹配越差，匹配的值越⼤。

  1. 相关匹配(CV_TM_CCORR)：利⽤模板与图像间的乘法进⾏匹配，数值越


⼤表示匹配程度较⾼，越⼩表示匹配效果差。

  1. 利⽤相关系数匹配(CV_TM_CCOEFF)：利⽤模板与图像间的相关系数匹


配，1表示完美的匹配，-1表示最差的匹配。

完成匹配后，使⽤cv.minMaxLoc()⽅法查找最⼤值所在的位置即可。如果使⽤平⽅

差作为⽐较⽅法，则最⼩值位置是最佳匹配位置。

示例：

在该案例中，载⼊要搜索的图像和模板，图像如下所示：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/72_1.jpg)

模板如下所示：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/72_2.jpg)

通过matchTemplate实现模板匹配，使⽤minMaxLoc定位最匹配的区域，并⽤矩形

标注最匹配的区域。
    
    
    import cv2 as cv
    
    import numpy as np
    
    from matplotlib import pyplot as plt
    
    # 1 图像和模板读取
    
    img = cv.imread('./image/wulin2.jpeg')
    
    template = cv.imread('./image/wulin.jpeg')
    
    h,w,l = template.shape
    
    # 2 模板匹配
    
    # 2.1 模板匹配
    
    res = cv.matchTemplate(img, template, cv.TM_CCORR)
    
    # 2.2 返回图像中最匹配的位置，确定左上⻆的坐标，并将匹配位置绘制在图像上
    
    min_val, max_val, min_loc, max_loc = cv.minMaxLoc(res)
    
    # 使⽤平⽅差时最⼩值为最佳匹配位置
    
    # top_left = min_loc
    
    top_left = max_loc
    
    bottom_right = (top_left[0] + w, top_left[1] + h)
    
    cv.rectangle(img, top_left, bottom_right, (0,255,0), 2)
    
    # 3 图像显示
    
    plt.imshow(img[:,:,::-1])
    
    plt.title('匹配结果'), plt.xticks([]), plt.yticks([])
    
    plt.show()

拓展：模板匹配不适⽤于尺度变换，视⻆变换后的图像，这时我们就要使⽤关键点

即可，这些关键点可以很好的处理尺度变化、视⻆变换、旋转变化、光照变化等，

霍夫变换常⽤来提取图像中的直线和圆等⼏何形状，如下图所示：

如果不⽌存在⼀条直线时，如下所示：

我们选择尽可能多的直线汇成的点，上图中三条直线汇成的A、B两点，将其对应

到这⾥我们似乎已经完成了霍夫变换的求解。但如果像下图这种情况时：

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/76_1.jpg)

上图中的直线是x = 2，那(k, q)怎么确定呢？

为了解决这个问题，我们考虑将笛卡尔坐标系转换为极坐标。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/76_2.jpg)

在极坐标下是⼀样的，极坐标中的点对应于霍夫空间的线，这时的霍夫空间是不在

是参数(k, q)的空间，⽽是(ρ, θ)的空间，ρ是原点到直线的垂直距离，θ表示直线的

垂线与横轴顺时针⽅向的夹⻆，垂直线的⻆度为0度，⽔平线的⻆度是180度。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/76_3.jpg)

我们只要求得霍夫空间中的交点的位置，即可得到原坐标系下的直线。

实现流程

假设有⼀个⼤⼩为100∗100的图⽚，使⽤霍夫变换检测图⽚中的直线，则步骤如下

所示：

直线都可以使⽤(ρ, θ) 表示，⾸先创建⼀个2D数组，我们叫做累加器，初始化

所有值为0，⾏表示ρ ，列表示θ 。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/77_1.jpg)

该数组的⼤⼩决定了结果的准确性，若希望⻆度的精度为1度，那就需要180

列。对于ρ，最⼤值为图⽚对⻆线的距离，如果希望精度达到像素级别，⾏数

应该与图像的对⻆线的距离相等。

取直线上的第⼀个点(x, y)，将其带⼊直线在极坐标中的公式中，然后遍历θ的

取值：0，1，2，...，180，分别求出对应的ρ值，如果这个数值在上述累加器

中存在相应的位置，则在该位置上加1.

取直线上的第⼆个点，重复上述步骤，更新累加器中的值。对图像中的直线上

的每个点都直线以上步骤，每次更新累加器中的值。

搜索累加器中的最⼤值，并找到其对应的(ρ, θ)，就可将图像中的直线表示出

来。

注意（CYC）：先二值化后，再进行霍夫变换。点的x和y就是二值化后为1的像素点的行索引和列索引，现在x和y已知，然后将累加器横轴每一个θ代入公式， ρ=\cos \theta + \sin \theta 得到对应的每一个ρ，ρ值对应累加器的位置+1。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/77_3.jpg)

#### 2.2 霍夫线检测

在OpenCV中做霍夫线检测是使⽤的API是：

cv.HoughLines(img, rho, theta, threshold)

参数：

img: 检测的图像，要求是⼆值化的图像，所以在调⽤霍夫变换之前⾸先要进⾏

⼆值化，或者进⾏Canny边缘检测

rho、theta: ρ 和θ的精确度

threshold: 阈值，只有累加器中的值⾼于该阈值时才被认为是直线。

#### 2.3 霍夫圆检测[了解]

  1. 原理


圆的表示式是

(x-a)^2+(y-b)^2 = r

其中a和b表示圆⼼坐标，r表示圆半径，因此标准的霍夫圆检测就是在这三个参数组成的三维空间累加器上进⾏圆形检测，此时效率就会很低，所以OpenCV中使⽤霍夫梯度法进⾏圆形的检测。

霍夫梯度法将霍夫圆检测范围两个阶段，第⼀阶段检测圆⼼，第⼆阶段利⽤圆⼼推导出圆半径。

圆⼼检测的原理：圆⼼是圆周法线的交汇处，设置⼀个阈值，在某点的相交的直线的条数⼤于这个阈值就认为该交汇点为圆⼼。

圆半径确定原理：圆⼼到圆周上的距离（半径）是相同的，确定⼀个阈值，只要相同距离的数量⼤于该阈值，就认为该距离是该圆⼼的半径。

原则上霍夫变换可以检测任何形状，但复杂的形状需要的参数就多，霍夫空间的维数就多，因此在程序实现上所需的内存空间以及运⾏效率上都不利于把标准霍夫变换应⽤于实际复杂图形的检测中。霍夫梯度法是霍夫变换的改进，它的⽬的是减⼩霍夫空间的维度，提⾼效率。

  1. API


在OpenCV中检测图像中的圆环使⽤的是API是：

circles = cv.HoughCircles(image, method, dp, minDist, param1=100, param2=...)

参数：

image：输⼊图像，应输⼊灰度图像

method：使⽤霍夫变换圆检测的算法，它的参数是

CV_HOUGH_GRADIENT

dp：霍夫空间的分辨率，dp=1时表示霍夫空间与输⼊图像空间的⼤⼩⼀

致，dp=2时霍夫空间是输⼊图像空间的⼀半，以此类推

minDist为圆⼼之间的最⼩距离，如果检测到的两个圆⼼之间距离⼩于该

值，则认为它们是同⼀个圆⼼

param1：边缘检测时使⽤Canny算⼦的⾼阈值，低阈值是⾼阈值的⼀半。

param2：检测圆⼼和确定半径时所共有的阈值

minRadius和maxRadius为所检测到的圆半径的最⼩值和最⼤值

返回：

circles：输出圆向量，包括三个浮点型的元素——圆⼼横坐标，圆⼼纵坐

标和圆半径

  1. 实现

## 轮廓检测

### 1 图像的轮廓

轮廓可以简单认为成将连续的点（连着边界）连在⼀起的曲线，具有相同的颜⾊或

者灰度。轮廓是图像⽬标的外部特征，这种特征对于我们进⾏图像分析，⽬标识别

和理解等更深层次的处理都有很重要的意义。

轮廓提取的基本原理：对于⼀幅背景为⿊⾊、⽬标为⽩⾊的⼆值图像，如果在图中

找到⼀个⽩⾊点，且它的8邻域（或4邻域）也均为⽩⾊，则说明该点是⽬标的内部

点，将其置为⿊⾊，视觉上就像内部被掏空⼀样；否则保持⽩⾊不变，该点是⽬标

的轮廓点。⼀般在寻找轮廓之前，都要将图像进⾏阈值化或Canny边缘检测，转换

为⼆值化图像。

在这⾥我们看下边缘提取和轮廓检测的区别：

边缘检测主要是通过⼀些⼿段检测数字图像中明暗变化剧烈（即梯度变化⽐较⼤）

像素点，偏向于图像中像素点的变化。如canny边缘检测，结果通常保存在和源图

⽚⼀样尺⼨和类型的边缘图中。

轮廓检测指检测图像中的对象边界，更偏向于关注上层语义对象。如OpenCV中的

findContours()函数， 它会得到每⼀个轮廓并以点向量⽅式存储，除此也得到⼀个

图像的拓扑信息，即⼀个轮廓的后⼀个轮廓、前⼀个轮廓等的索引编号。

#### 1.1 查找轮廓

在OPenCV中查找轮廓的API:

binary, contours, hierarchy = cv2.findContours(img, mode, method)

参数：

img: 输⼊图像，⼆值图

mode: 轮廓的检索模式，主要有四种⽅式：

cv2.RETR_EXTERNAL：只检测外轮廓，所有⼦轮廓被忽略

cv2.RETR_LIST：检测的轮廓不建⽴等级关系，所有轮廓属于同⼀等级

cv2.RETR_CCOMP：返回所有的轮廓，只建⽴两个等级的轮廓。⼀个对象的

外轮廓为第 1 级组织结构。⽽对象内部中空洞的轮廓为 第 2 级组织结构，空

洞中的任何对象的轮廓⼜是第 1 级组织结构。

cv2.RETR_TREE：返回所有的轮廓，建⽴⼀个完整的组织结构的轮廓。

method：轮廓的近似⽅法，主要有以下两种：

cv2.CHAIN_APPROX_NONE：存储所有的轮廓点，相邻的两个点的像素位置

差不超过1。

cv2.CHAIN_APPROX_SIMPLE：压缩⽔平⽅向，垂直⽅向，对⻆线⽅向的元

素，只保留该⽅向的终点坐标，例如⼀个矩形轮廓只需4个点来保存轮廓信

息。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/100_1.jpg)

返回：

binary: 返回的⼆值图像

contours: 检测出的轮廓，所有轮廓的列表结构，每个轮廓是⽬标对象边界点

的坐标的数组

hierarchy：轮廓的层次结构。

在检测轮廓时：有时对象可能位于不同的位置，也有可能⼀个形状在另外⼀个

形状的内部，这种情况下我们称外部的形状为⽗，内部的形状为⼦。按照这种

⽅式分类，⼀幅图像中的所有轮廓之间就建⽴⽗⼦关系。这样我们就可以确定

⼀个轮廓与其他轮廓是怎样连接的，⽐如它是不是某个轮廓的⼦轮廓，或者是

⽗轮廓。这种关系就是轮廓的层次关系。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/100_2.jpg)

在这幅图像中，我给这⼏个形状编号为 0-5。2 和 2a 分别代表最外边矩形的外

轮廓和内轮廓。

在这⾥边轮廓 0，1，2 在外部或最外边。我们可以称他们为 0 级，简单来说

就是他们属于同⼀级，接下来轮廓 2a，把它当成轮廓 2 的⼦轮廓。它就成为

第 1 级。轮廓 3 是轮廓 2a 的⼦轮廓，成为第 3 级。轮廓 3a 是轮廓 3 的⼦轮

廓，成为第 4 级,最后轮廓 4,5 是轮廓 3a 的⼦轮廓，成为5级，这样我们就构

建的轮廓的层级关系。

我们再回到返回值中，不管层次结构是什么样的， 每⼀个轮廓都包含⾃⼰的信

息。hierarchy使⽤包含四个元素的数组来表示：

[Next，Previous， First_Child，Parent]。

其中：

Next 表示同⼀级组织结构中的下⼀个轮廓，

以上图中的轮廓 0 为例，轮廓 1 就是他的 Next。同样，轮廓 1 的 Next 是 2，

Next=2。 那轮廓 2 呢？在同⼀级没有 Next。这时 Next=-1。⽽轮廓 4 的 Next

为 5，所以它的 Next=5。

Previous 表示同⼀级结构中的前⼀个轮廓。

轮廓 1 的 Previous 为轮廓 0，轮廓 2 的 Previous 为轮 廓 1。轮廓 0 没有

Previous，所以 Previous=-1。

First_Child 表示它的第⼀个⼦轮廓。

轮廓 2 的⼦轮廓为 2a。 所以它的 First_Child 为 2a。那轮廓 3a 呢？它有两个

⼦轮廓。但是我们只要第⼀个⼦轮廓，所以是轮 廓 4（按照从上往下，从左往

右的顺序排序）。

Parent 表示它的⽗轮廓。

与 First_Child 刚好相反。轮廓 4 和 5 的⽗轮廓是轮廓 3a。⽽轮廓 3a 的⽗轮

廓是 3。

注意：如果轮廓没有⽗轮廓或⼦轮廓时，则将其置为-1。

### 2 轮廓的特征

在提取了图像的轮廓后，可以计算轮廓的不同特征，我们现在主要看下：轮廓的⾯

积，周⻓，边界框等。

#### 2.1 轮廓⾯积

轮廓⾯积是轮廓所包围的区域的⾯积，在OpenCV中使⽤的API是：

area = cv.contourArea(cnt)

#### 2.2 轮廓周⻓

轮廓周⻓也被成为弧⻓，在OpenCV中使⽤的API是：

perimeter = cv2.arcLength(cnt,isclosed)

参数：

Isclosed: 指定轮廓的形状是闭合的（True），还是开放的

#### 2.3 轮廓近似

轮廓近似是将轮廓形状近似为到另外⼀种由更少点组成的轮廓形状，新轮廓的点的

数⽬由我们设定的准确度来决定。

假设我们要在⼀幅图像中查找⼀个矩形，然⽽这个图凹凸不平，直接提取轮廓⽆法

提取到⼀个完美的矩形。因此我们就可以使⽤轮廓近似函数来近似这个形状了。

![](http://cyc.chenyangstudy.cn/book_chenyang/a04opencv/images/104_1.jpg)

在OpenCV中使⽤的API是:

approx = cv.approxPolyDP(cnt,epsilon,isclosed)

参数：

cnt: 要进⾏轮廓近似的原始轮廓

epsilon:从原始轮廓到近似轮廓的最⼤距离，是⼀个准确度参数，该参数对调

整后的结果很重要。

Isclosed: 指定轮廓是否闭合

返回：

approx: 返回的点集，绘制时将其连接起来绘制最终的近似轮廓。

#### 2.5 边界矩形

轮廓检测中的边界矩形有两种，⼀种是直边界矩形，⼀种是旋转边界矩形，分别介

绍如下：

直边界矩形 ：⼀个直矩形，没有进⾏旋转。它不会考虑对象是否旋转，所以该边界

矩形的⾯积不是最⼩的。可以使⽤函数cv2.boundingRect()查找得到的。

x,y,w,h = cv2.boundingRect(cnt)

img = cv2.rectangle(img,(x,y),(x+w,y+h),(0,255,0),2)

返回值中，(x,y)是矩阵左上⻆的坐标，(w,h)是举⾏的宽和⾼。

旋转边⻆矩形 :这个边界矩形是⾯积最⼩的，他考虑了对象的旋转。⽤函数

cv2.minAreaRect(),返回的是⼀个Box2D结构，其中包含矩形左上⻆⻆点的坐标

(x,y)，以及矩形的宽和⾼(w,h)，以及旋转⻆度。但是要绘制这个矩形需要矩形的4

个⻆点。可以通过函数cv2.boxPoints()获得。

s = cv2.minAreaRect(cnt)

a = cv2.boxPoints(s)

a = np.int0(a)#必须转换a的类型为int型

cv2.polylines(im,[a],True,(0,0,255),3)

#### 2.7 椭圆拟合

椭圆拟合法的基本思路是：对于给定平⾯上的⼀组样本点，寻找⼀个椭圆，使其尽

可能接近这些样本点。也就是说，将图像中的⼀组数据以椭圆⽅程为模型进⾏拟

合，使某⼀椭圆⽅程尽量满⾜这些数据，并求出该椭圆⽅程的各个参数。

就椭圆拟合⽽⾔，就是先假设椭圆参数，得到每个待拟合点到该椭圆的距离之和，

也就是点到假设椭圆的误差，求出使这个和最⼩的参数。

在OPenCV中我们使⽤cv2.ellipse()来进⾏椭圆拟合，将边界矩形中的代码改为如下

所示，就可得到椭圆拟合的结果：

ellipse = cv.fitEllipse(cnt)

img = cv.ellipse(img,ellipse,(0,255,0),2)

#### 2.8 直线拟合

直线拟合就是将图像中的对象拟合成⼀条直线过程，在OPenCV中拟合直线的API

是：

output = cv2.fitLine(points, distType, param, reps, aeps)

参数：

points: 待拟合直线的点的集合，可以是检测处理轮廓结果

distype: 距离公式，在进⾏拟合是，要使输⼊点到拟合直线的距离之和最⼩，

常⽤的⽤以下⼏种：

cv2.DIST_L1: 曼哈顿距离

cv2.DIST_L2: 欧式距离

cv2.DIST_C:切⽐雪夫距离

param: 距离参数，可以设为0

Reps,aeps:⽤于表示拟合曲线所需要的径向和⻆度精度，通常设为0.01

返回：

output: [vx,vy,x,y]的1*4 的数组，前两个表示直线的⽅向，即vy/vx表示斜率，

后两位表示直线上的⼀个点。

将边界矩形中的代码改为如下示，即可进⾏直线拟合：

rows,cols = img.shape[:2]

[vx,vy,x,y] = cv.fitLine(cnt, cv2.DIST_L2,0,0.01,0.01)

lefty = int((-x*vy/vx) + y)

righty = int(((cols-x)*vy/vx)+y)

im = cv.line(img,(cols-1,righty),(0,lefty),(0,255,0),2)

### 3 图像的矩特征

矩函数在图像分析中有着⼴泛的应⽤，如模式识别、⽬标分类、⽬标识别与⽅位估

计、图像的编码与重构等。从⼀幅图像计算出来的矩集，不仅可以描述图像形状的

全局特征，⽽且可以提供⼤量关于该图像不同的⼏何特征信息，如⼤⼩，位置、⽅

向和形状等。

#### 3.1 矩的概念

矩是概率与统计中的⼀个概念，是随机变量的⼀种数字特征。矩的定义如下：

#### 3.2 图像中的矩特征

对于⼀幅图像，我们把像素的坐标看成是⼀个⼆维随机变量(𝑋,𝑌)，那么⼀幅灰度图

像可以⽤⼆维灰度密度函数来表示，因此可以⽤矩来描述灰度图像的特征。

moments(array, binaryImage=false )

参数：

array:输⼊数组，也可以是灰度图像，也可以是⼆维数组，例如提取的轮廓结果。

BinaryImage:默认是false，若为True，则所有⾮零的像素都会按值1对待，也 就是说相当于对图像进⾏了⼆值化处理，阈值为1，此参数仅对图像有效。

返回：

moment: 返回数组的中⼼矩

计算Hu矩时，将中⼼距输⼊即可。

示例：

计算上⼀章节中箭头的矩特征，代码如下所示： 
    
    
    import numpy as np
    
    import cv2 as cv
    
    import matplotlib.pyplot as plt
    
    # 1 图像读取
    
    img = cv.imread('./image/arrows.jpg')
    
    imgray = cv.cvtColor(img,cv.COLOR_BGR2GRAY)
    
    # 2 计算图像的Hu矩
    
    imgmn = cv.moments(imgray)
    
    imghu = cv.HuMoments(imgmn)
    
    print("图像Hu矩结果：n",imghu)
    
    # 3 计算轮廓的Hu矩
    
    # 3.1 转换为⼆值图
    
    ret,thresh = cv.threshold(imgray,127,255,0)
    
    # 3.2 轮廓提取
    
    image, contours, hierarchy = cv.findContours(thresh,1,2)
    
    # 3.3 计算轮廓的Hu矩
    
    cnt = contours[1]
    
    mn = cv.moments(cnt)
    
    hu = cv.HuMoments(mn)
    
    print("Hu矩结果：n",hu)
