### 2 随机森林构造过程

在机器学习中， **随 机森林是一个包含多个决策树的分类器** ，并且其输出的类别是由个别树输出的类别的众数而定。

**随 机森林** **= Bagging +** **决 策树**

![image-20190214160355272](http://cyc.chenyangstudy.cn/book_chenyang/a02机器学习/images/tf1.png)

例如, 如果你训练了5个树, 其中有4个树的结果是True, 1个树的结果是False, 那么最终投票结果就是True

**随 机森林够造过程中的关键步骤** （用N来表示训练用例（样本）的个数，M表示特征数目）：

​ **1 ）一次随机选出一个样本，有放回的抽样，重复N次（有可能出现重复的样本）**

​ **2 ） 随机去选出m个特征, m <

  * **思 考**

    * 1.为什么要随机抽样训练集？


如果不进行随机抽样，每棵树的训练集都一样，那么最终训练出的树分类结果也是完全一样的
    
    
    * 2.为什么要有放回地抽样？
    

如果不是有放回的抽样，那么每棵树的训练样本都是不同的，都是没有交集的，这样每棵树都是"有偏的"，都是绝对"片面的"（当然这样说可能不对），也就是说每棵树训练出来都是有很大的差异的；而随机森林最后分类取决于多棵树（弱分类器）的投票表决。

### 3 随机森林api介绍

  * sklearn.ensemble.RandomForestClassifier(n_estimators=10, criterion='gini', max_depth=None, bootstrap=True, random_state=None, min_samples_split=2)
    * n_estimators：integer，optional（default = 10）森林里的树木数量120,200,300,500,800,1200
    * Criterion：string，可选（default ="gini"）分割特征的测量方法
    * max_depth：integer或None，可选（默认=无）树的最大深度 5,8,15,25,30
    * max_features="auto",每个决策树的最大特征数量
      * If "auto", then `max_features=sqrt(n_features)`.
      * If "sqrt", then `max_features=sqrt(n_features)`(same as "auto").
      * If "log2", then `max_features=log2(n_features)`.
      * If None, then `max_features=n_features`.
    * bootstrap：boolean，optional（default = True）是否在构建树时使用放回抽样
    * min_samples_split:节点划分最少样本数
    * min_samples_leaf:叶子节点的最小样本数
  * 超参数：n_estimator, max_depth, min_samples_split,min_samples_leaf

**bagging 集成与boosting集成的区别：**

区别一:数据方面

Bagging：对数据进行采样训练；

Boosting：根据前一轮学习结果调整数据的重要性。

区别二:投票方面

Bagging：所有学习器平权投票；

Boosting：对学习器进行加权投票。

区别三:学习顺序

Bagging的学习是并行的，每个学习器没有依赖关系；

Boosting学习是串行，学习有先后顺序。

区别四:主要作用

Bagging主要用于提高泛化性能（解决过拟合，也可以说降低方差）

Boosting主要用于提高训练精度 （解决欠拟合，也可以说降低偏差）

### 2 GBDT(了解)

梯度提升决策树(GBDT Gradient Boosting Decision Tree) **是 一种迭代的决策树算法** ， **该 算法由多棵决策树组成，所有树的结论累加起来做最终答案。** 它在被提出之初就被认为是泛化能力（generalization)较强的算法。近些年更因为被用于搜索排序的机器学习模型而引起大家关注。

**GBDT = 梯度下降 + Boosting + 决策树**

1.使用梯度下降法优化代价函数；

### 3.XGBoost【了解】

**XGBoost= 二阶泰勒展开+boosting+决策树+正则化**

  * **面 试题：了解XGBoost么，请详细说说它的原理**


回答要点：二阶泰勒展开，boosting，决策树，正则化

**Boosting** ：XGBoost使用Boosting提升思想对多个弱学习器进行迭代式学习

**二 阶泰勒展开** ：每一轮学习中，XGBoost对损失函数进行二阶泰勒展开，使用一阶和二阶梯度进行优化。

**决 策树** ：在每一轮学习中，XGBoost使用决策树算法作为弱学习进行优化。

**正 则化** ：在优化过程中XGBoost为防止过拟合，在损失函数中加入惩罚项，限制决策树的叶子节点个数以及决策树叶子节点的值。

## 4\. 总结
    
    
    1、投票分类器
    将不同的简单的机器学习模型的预测结合起来最简单的办法，基于各个模型的预测，给出平均预测结果。
    2、bagging
    自动抽取样本，n个弱学习器中选择投票数最高的或者回归结果进行算术平均得到的值。
    3、Random_Forest
    bagging算法的优化，自助抽样，弱学习器，随机选择样本特征，增加泛化能力，
    n个弱学习器中选择投票数最高的或者回归结果进行算术平均得到的值。
    4、Adaboost（自适应增强）
    非自助抽样样本，初始化权重，若学习器。
    用学习器的误差来更新学习器权重，用样本误差更新样本权重。用平均加权法得到最终分类模型或者回归模型。
    5、GDBT
    GDBT中的树都是回归树，不是分类数。非自助抽取样本，学习器，每个分类器基于上一轮分类器的残差做训练，需要
    累加所有树的结果，所以只能通过回归来完成。
    优点：不需要做复杂的特征工程和特征变化，能灵活处理各种类型的数据。
    6、Xgboost
    与GDBT区别：
    1、代价函数使用二阶泰勒展开。
    2、loss中加入了正则。
    优点：速度更快，内存消耗更低。

    game_date   比赛日期
    matchup 对阵双方
    opponent    对手
    game_id 比赛的编号
    shot_id 出手的编号
    """
    
    raw = pd.read_csv("data.csv")
    # print(raw.shape)  # 查看数据形状 (30697, 25)
    # print(raw.head())
    # print(raw["shot_made_flag"].count()) #shot_made_flag 是否命中 25697,30697-25697剩下的5000条空的数据就是预测集
    kobe = raw[pd.notnull(raw["shot_made_flag"])]
    kobe_predict = raw[pd.isnull(raw["shot_made_flag"])]
    # print(kobe.shape)  # (25697, 25)
    # print(kobe.columns)
    # todo: 分析出手的位置
    # loc_x 出手的x坐标
    # loc_y 出手的y坐标
    plt.figure(figsize=(10, 10))
    plt.subplot(121)
    plt.title("loc_x and loc_y", fontsize=16)
    plt.scatter(kobe["loc_x"], kobe['loc_y'], color='red', alpha=0.2)
    # lon   出手的经度
    # lat   出手的纬度
    plt.subplot(122)
    plt.title("lon and lat", fontsize=16)
    plt.scatter(kobe["lon"], kobe['lat'], color='green', alpha=0.2)
    plt.show()
    
    # todo: 出手的距离和角度
    raw["dist"] = np.sqrt(raw['loc_x'] ** 2 + raw["loc_y"] ** 2)  # 新增一列，与篮筐的距离
    # 角度
    loc_x_zero = raw['loc_x'] == 0  # 正对篮筐的角度的数据索引
    raw["angle"] = np.array([0] * len(raw))
    # ~取反
    raw['angle'][~loc_x_zero] = np.arctan(raw['loc_y'][~loc_x_zero] / raw['loc_x'][~loc_x_zero])  # 弧度
    raw["angle"][loc_x_zero] = np.pi / 2  # π/2  90
    
    # print(raw['dist'].describe())
    # print(raw['angle'].describe())
    
    # todo: 新增一列，出手时间点,距离比赛结束(每一节12分钟结束)剩多少秒
    raw["ramaining_time"] = raw["minutes_remaining"] * 60 + raw["seconds_remaining"]
    # print(raw["ramaining_time"].describe())
    
    #todo:action_type 动作类型
    # print(len(raw["action_type"].unique()))  #57种攻击模式
    # print(raw["action_type"].value_counts())  #科比最喜欢的还是跳投 Jump Shot  上篮 Layup Shot
    
    #todo:combined_shot_type组合投篮类型：Jump Shot跳投，Layup上篮 扣篮Dunk，勾手Tip Shot，擦板Hook Shot，罚球Bank Shot
    # print(len(raw["combined_shot_type"].unique()))  #6种组合投篮类型，
    # print(raw["combined_shot_type"].value_counts())  #科比最喜欢的还是跳投 Jump Shot  勾手 Layup Shot 扣篮dunk shot是262次
    
    #todo:shot_type投篮类型 2PT Field Goal 3PT Field Goal
    # print(len(raw["shot_type"].unique()))  #2分球和3分球
    # print(raw["shot_type"].value_counts())  #科比5次投篮出手一次三分
    
    #todo:season赛季
    # print(raw["season"].unique())  #20个赛季
    raw["season"] = raw["season"].apply(lambda i:int(i.split("-")[-1]))
    # print(raw["season"].value_counts())
    
    #todo： team_id team_name 球队编号和名字
    # print(kobe["team_id"].unique()) #[1610612747]
    # print(kobe["team_name"].unique()) #['Los Angeles Lakers']
    
    #todo:出手区域分析及可视化
    # plt.figure(figsize=(5,5))
    # plt.scatter(raw["dist"],raw["shot_distance"],color='blue')
    # plt.title('dist and shot_distance') #我们计算的dist和给的shot_distance相等
    # plt.show()
    gs = kobe.groupby('shot_zone_area') #根据出手区域划分
    # print(gs['shot_zone_area'].value_counts())
    """       
    Back Court(BC)         Back Court(BC)              72
    Center(C)              Center(C)                11289
    Left Side Center(LC)   Left Side Center(LC)      3364
    Left Side(L)           Left Side(L)              3132
    Right Side Center(RC)  Right Side Center(RC)     3981
    Right Side(R)          Right Side(R)             3859
    """
    # shot_zone_area    出手区域，左侧，右侧，中场，后场等
    # shot_zone_basic   另一种划分出手区域的方式，中线，禁区，油漆区，左侧底角，右侧底角等
    # shot_zone_range   出手区域的距离，小于8英尺，8-16英尺，16-24英尺，24英尺以上等  1英尺大概0.3米
    def scatter_plot_by_category(feat='shot_zone_area'):
        gs = kobe.groupby(feat)
        cs = cm.jet(np.linspace(0,1,len(gs))) #获取颜色集
        for g,c in zip(gs,cs):
            plt.scatter(g[1]['loc_x'],g[1]["loc_y"],color=c,alpha=0.1)
    
    plt.figure(figsize=(24,8))
    plt.subplot(131)
    plt.title("shot_zone_area")
    scatter_plot_by_category(feat='shot_zone_area')
    plt.subplot(132)
    plt.title("shot_zone_basic")
    scatter_plot_by_category(feat='shot_zone_basic')
    plt.subplot(133)
    plt.title("shot_zone_range")
    scatter_plot_by_category(feat='shot_zone_range')
    plt.show()
    
    #############################################################################
    #对全部的数据剔除一些列，后续进行建模
    # print(raw.columns)
    """
    Index(['action_type', 'combined_shot_type', 'game_event_id', 'game_id', 'lat',
           'loc_x', 'loc_y', 'lon', 'minutes_remaining', 'period', 'playoffs',
           'season', 'seconds_remaining', 'shot_distance', 'shot_made_flag',
           'shot_type', 'shot_zone_area', 'shot_zone_basic', 'shot_zone_range',
           'team_id', 'team_name', 'game_date', 'matchup', 'opponent', 'shot_id',
           'dist', 'angle', 'ramaining_time'],
          dtype='object')
    """
    drops = ['shot_id','team_id','team_name','shot_zone_area','shot_zone_basic','shot_zone_range',
             'matchup','lon','lat','minutes_remaining','seconds_remaining','shot_distance','loc_x','loc_y','game_event_id',
             'game_id','game_date']
    for drop in drops:
        raw = raw.drop(drop,axis=1) #删除列
    
    # print(raw.head())
    
    
    #todo:数据预处理，投篮方式进行独热编码
    ohehot_columns = ['action_type','combined_shot_type','shot_type','opponent','period','season']
    for var in ohehot_columns:
        raw = pd.concat([raw,pd.get_dummies(raw[var],prefix=var)],axis=1) #横向拼接独热编码后的数组，prefix是独热后的列索引开头字符
        raw = raw.drop(var,axis=1) #删除调独热编码前的特征列，有独热编码即可
    # print(raw.head())
    # print(len(raw.columns)) #136
    
    
    #todo: 训练集
    train_kobe = raw[pd.notnull(raw["shot_made_flag"])] #非空的就是训练集
    train_label = train_kobe["shot_made_flag"]
    train_kobe = train_kobe.drop("shot_made_flag",axis=1)
    
    #todo: 测试集
    test_kobe = raw[pd.isnull(raw["shot_made_flag"])] #空的就是测试集
    test_kobe = test_kobe.drop("shot_made_flag",axis=1)
    
    #训练模型，寻找最佳参数
    max_score = 0
    best_n = 0
    score_n = []
    range_n = np.logspace(0,2,num=3).astype(int) #[  1  10 100]  等比数列10^0  10^1  10^2
    for n in range_n:
        print(f"当前迭代的参数n_estimators={n}")
        rfc_score = 0
        rfc = RandomForestClassifier(n_estimators=n)
        kfold = ms.KFold(n_splits=10,random_state=2023)
        cv_result = ms.cross_val_score(rfc,train_kobe,train_label,cv=kfold,scoring='accuracy')
        rfc_score = cv_result.mean()
        score_n.append(rfc_score)
        if rfc_score > max_score:
            max_score = rfc_score
            best_n = n
    
    print(f"最佳参数n_estimators={best_n},最佳分数：{max_score}")
    
    model = RandomForestClassifier(n_estimators=best_n)
    model.fit(train_kobe,train_label)
    pred_y = model.predict(test_kobe)
    kobe_predict["shot_made_flag"] = pred_y.ravel()
    kobe_predict.to_csv("测试集完成预测.csv",index=False)
